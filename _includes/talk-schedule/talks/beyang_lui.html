How we test a large-scale Go web app

Friday 20th February, 2015    3:10 pm to 3:35 pm (IST)

Nearly every programmer agrees that testing is important. But tests can also be slow to run or annoying to write. I'll discuss how we've structured the tests for Sourcegraph, a large-scale Go web app. I'll talk about the different types of tests we have, from unit to integration tests, and share some patterns we've found useful for writing tests that are concise, reliable, and fast.

Beyang Liu @beyang
Beyang is co-creator of Sourcegraph, an intelligent code search engine built with Go. Before Sourcegraph, he worked on data analysis and visualization software at Palantir and conducted research in computer vision and machine learning at Stanford University. He also plays the fiddle and lives in San Francisco.
